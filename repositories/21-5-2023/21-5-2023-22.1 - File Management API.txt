Nama Proyek: File Management API

Deskripsi:
Proyek ini bertujuan untuk membangun sebuah API backend yang dapat mengelola file. API ini akan menyediakan fitur-fitur berikut:

Upload File: API akan memiliki endpoint untuk mengunggah file ke server. File yang diunggah akan disimpan di server dan informasi 
file akan disimpan dalam database (misalnya, nama file, tipe file, ukuran file, dan timestamp unggah).

Download File: API akan menyediakan endpoint untuk mengunduh file berdasarkan ID atau nama file. Pengguna dapat mengakses file yang 
telah diunggah sebelumnya melalui endpoint ini.

Update File: API akan memiliki endpoint untuk memperbarui file yang telah diunggah. Pengguna dapat mengganti file yang ada dengan 
file baru, dan informasi file yang terkait akan diperbarui di database.

Delete File: API akan menyediakan endpoint untuk menghapus file yang telah diunggah. Saat pengguna mengirim permintaan penghapusan, 
file dan informasi terkaitnya akan dihapus dari server dan database.

List Files: API akan memiliki endpoint untuk menampilkan daftar file yang telah diunggah. Pengguna dapat memperoleh informasi dasar 
tentang file-file yang ada, seperti nama file, tipe file, ukuran file, dan timestamp unggah.

Teknologi yang Disarankan:

Bahasa pemrograman: Node.js (dengan Express.js framework)
Database: MongoDB atau PostgreSQL (tergantung preferensi dan kebutuhan proyek)
Sistem Penyimpanan File: Simpan file-file di server lokal atau gunakan penyimpanan berbasis cloud seperti AWS S3 atau Google Cloud Storage.
Pengujian (Testing): Mocha, Jest, atau alat pengujian lainnya yang sesuai dengan preferensi Anda.
Langkah-langkah Implementasi:

Buat proyek Node.js menggunakan Express.js framework.
Konfigurasi database dan sambungkan dengan proyek.
Tentukan model untuk entitas File dan buat skema database yang sesuai.
Buat API endpoint untuk setiap fitur yang disebutkan di atas (unggah, unduh, perbarui, hapus, dan daftar file).
Inisialisasi git.
Implementasikan logika pemrosesan file untuk setiap endpoint.
Buat pengujian untuk memastikan fungsi-fungsi API bekerja dengan benar.
Uji API menggunakan Postman atau alat serupa untuk memverifikasi kebenaran implementasi.
Publikasikan proyek ke server yang sesuai.
